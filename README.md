# Business Minutes Builder（議事録下書き作成ツール）

会議メモや録音データから作成した文字起こしを入力すると、ビジネス用途で使いやすい議事録フォーマットへ整理する静的Webアプリです。HTML / CSS / JavaScriptのみで動作し、外部ライブラリ、サーバー、API連携は不要です。

このツールの目的は、単なる文字起こし整理ではありません。会議参加者や上長が後から読んだときに、「誰が・何を・いつまでに・なぜ行うか」をすばやく把握できる状態を目指します。

## ツール概要

- 左側に会議情報と文字起こしの入力欄、右側に議事録出力欄を配置した2カラムUIです。
- PCでは横並び、スマートフォンでは縦並びで表示されます。
- 入力された会議メモをルールベースで解析し、以下を抽出します。
  - 決定事項
  - TODO候補
  - 未決事項・課題
  - 次回会議までのアクション
  - 担当者候補
  - 期限候補
  - 優先度（高・中・低）
- WordやOutlookに貼り付けやすいテキスト形式で出力します。
- GitHub Pagesにそのまま公開できます。

## 使い方

1. `index.html` をブラウザで開きます。
2. 左側の入力欄に会議情報、会議目的、アジェンダ、会議メモ・文字起こしを入力します。
3. `議事録を作成` を押すと、右側に議事録下書きが生成されます。
4. 必要に応じて `議事録をコピー` または `テキストでダウンロード` を使用します。
5. より高精度な要約にしたい場合は、`ChatGPT貼り付け用プロンプト生成` を押し、生成されたプロンプトをChatGPTに貼り付けて仕上げます。
6. 入力例を確認したい場合は、`サンプル入力` を押すと物流業界の会議メモ例が自動入力されます。

## 入力項目の説明

| 項目 | 説明 |
| --- | --- |
| 会議名 | 議事録のタイトルになる会議名称です。 |
| 会議日時 | 会議の実施日時です。テキスト入力のため任意形式で入力できます。 |
| 参加者 | 社内外の参加者、部署、役割を入力します。 |
| クライアント名または案件名 | 顧客名、案件名、プロジェクト名を入力します。 |
| 会議目的 | 会議で何を決める／確認するかを入力します。 |
| 会議アジェンダ | 議題を番号付きまたは改行区切りで入力します。 |
| 会議メモ・文字起こし内容 | 録音文字起こしや手元メモを貼り付けます。 |
| 重要キーワード | B/L、ETA、Free Timeなど、議事録で意識したい語句を入力します。 |
| 機密情報レベル | `社外秘`、`社内限り`、`一般共有可` から選択します。 |
| 出力トーン | `フォーマル`、`簡潔`、`詳細` から選択します。 |
| 用途 | `社内共有用`、`クライアント提出用`、`経営層報告用` から選択します。 |

## 議事録出力フォーマット

出力は以下の構成で生成されます。

```text
【会議議事録】

■エグゼクティブサマリー
■会議概要
■アジェンダ
■議論内容サマリー
■決定事項
■未決事項・課題
■TODO一覧
■次回会議までのアクション
■要確認事項
■補足事項
```

TODO一覧は、`No / タスク内容 / 担当者 / 期限 / 優先度 / 補足` の表形式で出力します。担当者や期限が読み取れない場合は `要確認` として明示します。

## 整理ロジック

JavaScript内で以下のキーワードをもとに分類します。

- `決定`、`合意`、`進める`、`承認`、`確定` など：決定事項
- `対応`、`作成`、`確認`、`連絡`、`提出`、`共有`、`修正` など：TODO候補
- `課題`、`懸念`、`問題`、`リスク`、`未確認`、`要確認` など：未決事項・課題
- `次回`、`次の会議`、`次回まで` など：次回会議までのアクション
- `担当`、`〇〇さん`、`〇〇部` など：担当者候補
- `明日`、`今週中`、`来週`、`月末`、`MM/DD` など：期限候補

優先度は、期限が近いもの、顧客影響があるもの、意思決定に関わるものを `高`、通常対応を `中`、参考情報や余裕のある確認事項を `低` として分類します。

## ChatGPT貼り付け用プロンプト生成機能

このアプリ単体では完全なAI要約は行いません。以下の2段階方式で利用することを想定しています。

### ステップ1：アプリ内で下書き整理

ブラウザ上で入力テキストをルールベースにより分類し、議事録フォーマットの下書きを作成します。

### ステップ2：ChatGPTで高精度化

`ChatGPT貼り付け用プロンプト生成` を押すと、入力情報と議事録作成指示をまとめた完成版プロンプトを生成します。プロンプトには以下の役割・条件を含めています。

- 外資系コンサルティングファーム出身のPM兼ビジネスアナリストとして整理する
- 決定事項、TODO、未決事項、期限、担当者を明確化する
- A4 1〜2枚程度の議事録にする
- 雑談、挨拶、オンライン会議特有のノイズを除外する
- 不明点は要確認事項として整理する

生成されたプロンプトをChatGPTに貼り付けることで、API連携なしでも高精度な議事録作成に進めます。

## GitHub Pagesで公開する方法

1. このリポジトリをGitHubにpushします。
2. GitHubのリポジトリ画面で `Settings` を開きます。
3. 左側メニューから `Pages` を選択します。
4. `Build and deployment` の `Source` を `Deploy from a branch` にします。
5. 公開するブランチを選び、フォルダは `/ (root)` を選択します。
6. `Save` を押すと、数分後にGitHub PagesのURLが表示されます。

## ローカル確認

リポジトリ直下で以下を実行すると、ローカルサーバーで確認できます。

```bash
python3 -m http.server 4173 --bind 127.0.0.1
```

ブラウザで `http://127.0.0.1:4173/` を開いてください。
